# 筋トレ記録アプリ 要件定義書

## 1. プロジェクト概要

筋トレの記録を効率化するWebアプリケーション。PWA対応し、スマホでもネイティブアプリのように使える。
現状メモ帳で行っているトレーニング記録の課題を解決し、記録・振り返り・共有をスムーズにする。

### 1.1 背景・課題

- トレーニング時に毎回メモ帳に記入しているが、記入が煩わしい
- ABC等の複数メニューをローテーションしており、その日にやる種目は決まっているのに毎回書き直している
- 前回のトレーニングとの差分（重量・回数の変化）が見えづらい
- 前回の記録をコピーしてから記入を始めるが、種目を削ったり書き足したりする手間がかかる
- トレーニング後にメモをトレーナーに送る際、コピー操作が煩わしい

## 2. 技術スタック

- フロントエンド: Next.js App Router + TypeScript
- インフラ: Vercel
- データベース: Prisma + PostgreSQL
- 認証: Supabase Authentication
- パッケージマネージャー: pnpm

## 3. ドメインモデル

### 3.1 ユーザー（User）

アプリケーションの利用者。メールアドレスで一意に識別される。論理削除に対応する。

### 3.2 登録種目（Exercise）

自分が取り組む種目のマスタデータ。種目名・重量・回数・セット数を持つ。
トレーニング記録をつけた際に最新の数値（重量・回数）が更新される。論理削除に対応する。

### 3.3 登録メニュー（Menu）

トレーニングメニューのテンプレート。タイトル（例: 胸肩メニュー）を持つ。

### 3.4 メニュー種目（MenuItems）

登録メニューと登録種目の中間テーブル。メニューに含まれる種目の一覧を管理する。

### 3.5 トレーニング記録（Training）

実際のトレーニング記録。トレーニング日時（trained_at）と、ベースにしたメニューID（base_menu_id）、通算トレーニング回数（training_count）を持つ。

### 3.6 種目記録（ExerciseLog）

トレーニング記録内の各種目の記録。トレーニング記録と登録種目を参照する。

### 3.7 セット記録（ExerciseSet）

種目記録内の各セットの記録。セット番号（set_count）・重量（weight）・回数（reps）を持つ。

### 3.8 リレーション

- User : Exercise = 1 : N（ユーザーが種目を作成）
- User : Menu = 1 : N（ユーザーがメニューを所有）
- User : Training = 1 : N（ユーザーがトレーニングを実施）
- Menu : MenuItems = 1 : N（メニューが種目を含む）
- MenuItems : Exercise = N : 1（メニュー種目が登録種目を参照）
- Training : ExerciseLog = 1 : N（トレーニング記録が種目記録を持つ）
- Exercise : ExerciseLog = 1 : N（登録種目が種目記録で参照される）
- ExerciseLog : ExerciseSet = 1 : N（種目記録がセット記録を持つ）

## 4. 機能要件

### 4.1 認証機能

#### FR-AUTH-001: Googleログイン

| 項目 | 内容 |
|------|------|
| 概要 | Supabase AuthenticationによるGoogleアカウントログイン |
| 優先度 | 必須 |

**詳細要件:**
- Supabase Authenticationを使用したGoogleアカウントでのログイン機能を提供する
- 未ログイン状態ではログイン画面のみを表示する
- ログイン成功後、自動的にホーム画面へ遷移する
- ログアウト機能を提供し、セッションを完全にクリアする

#### FR-AUTH-002: セッション管理

| 項目 | 内容 |
|------|------|
| 概要 | ログイン状態の永続化と管理 |
| 優先度 | 必須 |

**詳細要件:**
- ログイン状態をブラウザセッションで維持する
- セッション有効期限は30日間とする
- 複数デバイスからの同時ログインを許可する
- Next.js App Routerのミドルウェアで認証状態を検証する

### 4.2 種目登録機能

#### FR-EXERCISE-001: 種目の作成

| 項目 | 内容 |
|------|------|
| 概要 | トレーニング種目をマスタデータとして登録する |
| 優先度 | 必須 |

**入力フィールド仕様:**

| フィールド名 | 必須/任意 | データ型 | バリデーション |
|-------------|----------|---------|---------------|
| 種目名 (name) | **必須** | string | 1文字以上、50文字以下 |
| 重量 (weight) | **必須** | int | 0以上、999以下 |
| 回数 (reps) | **必須** | int | 1以上、100以下（整数） |
| セット数 (sets) | **必須** | int | 1以上、20以下（整数） |

**処理フロー:**
1. ユーザーが種目情報を入力
2. クライアント側でZodによるバリデーション
3. データベースに種目データを保存
4. 成功後、種目一覧を更新表示

#### FR-EXERCISE-002: 種目の編集

| 項目 | 内容 |
|------|------|
| 概要 | 登録済み種目の情報を編集する |
| 優先度 | 必須 |

**詳細要件:**
- 種目名、重量、回数、セット数を編集可能とする
- 更新日時（updatedAt）を自動更新する
- 編集はインライン編集またはモーダルで操作する

#### FR-EXERCISE-003: 種目の削除

| 項目 | 内容 |
|------|------|
| 概要 | 登録済み種目を削除する |
| 優先度 | 必須 |

**詳細要件:**
- 確認ダイアログを表示後、種目を削除する
- メニューに紐づいている種目は削除できない（エラーメッセージを表示）
- 削除は論理削除とする（is_deletedフラグをtrueに設定）

#### FR-EXERCISE-004: 種目一覧表示

| 項目 | 内容 |
|------|------|
| 概要 | 登録済み種目の一覧を表示する |
| 優先度 | 必須 |

**詳細要件:**
- 登録済みの全種目（is_deleted=falseのもの）を一覧表示する
- 各種目は種目名、重量、回数、セット数を表示する
- 種目名で検索・フィルタが可能

#### FR-EXERCISE-005: 種目の自動登録

| 項目 | 内容 |
|------|------|
| 概要 | メニュー作成時・トレーニング記録時に新規種目を自動登録する |
| 優先度 | 必須 |

**詳細要件:**
- メニュー作成画面またはトレーニング記録画面で新規種目を入力した場合、種目マスタにも自動的に登録する
- 既存の種目名と重複する場合は新規登録せず既存の種目を使用する

### 4.3 メニュー作成機能

#### FR-MENU-001: メニューの作成

| 項目 | 内容 |
|------|------|
| 概要 | トレーニングメニューのテンプレートを作成する |
| 優先度 | 必須 |

**入力フィールド仕様:**

| フィールド名 | 必須/任意 | データ型 | バリデーション |
|-------------|----------|---------|---------------|
| メニュータイトル (title) | **必須** | string | 1文字以上、50文字以下 |
| 種目リスト (exercises) | **必須** | ExerciseItem[] | 1件以上 |

**種目リスト（MenuItems）の各項目:**

| フィールド名 | 必須/任意 | データ型 | バリデーション |
|-------------|----------|---------|---------------|
| 種目ID (exercise_id) | **必須** | string | 登録済み種目のID |

**備考:** セット数・回数・重量はMenuItemsには持たず、参照先のExerciseの値を使用する。

**処理フロー:**
1. ユーザーがメニュータイトルを入力
2. 登録済み種目から選択、または新規種目を入力して追加
3. 種目の並び順をドラッグ&ドロップで変更可能
4. クライアント側でZodによるバリデーション
5. データベースにMenuとMenuItemsを保存
6. 成功後、メニュー一覧を更新表示

**備考:**
- ABCなど複数メニューをローテーションする運用を想定
- 全画面モーダルで作成操作を行う

#### FR-MENU-002: メニューの編集

| 項目 | 内容 |
|------|------|
| 概要 | 登録済みメニューのテンプレートを編集する |
| 優先度 | 必須 |

**詳細要件:**
- メニュータイトル、種目リストの全項目を編集可能とする
- 種目の追加・削除・並び替えが可能
- 更新日時（updatedAt）を自動更新する
- 全画面モーダルで編集操作を行う

#### FR-MENU-003: メニューの削除

| 項目 | 内容 |
|------|------|
| 概要 | 登録済みメニューを削除する |
| 優先度 | 必須 |

**詳細要件:**
- 確認ダイアログを表示後、メニューを削除する
- 過去のトレーニング記録には影響しない（メニューはテンプレートとして使うのみのため）
- 削除は物理削除とする

#### FR-MENU-004: メニュー一覧表示

| 項目 | 内容 |
|------|------|
| 概要 | 登録済みメニューの一覧を表示する |
| 優先度 | 必須 |

- 各メニューはタイトル、含まれる種目数、更新日時を表示する
- メニューカードをタップすると編集モーダルを表示する

### 4.4 トレーニング記録機能

#### FR-TRAINING-001: トレーニング記録の作成

| 項目 | 内容 |
|------|------|
| 概要 | メニューをテンプレートにしてトレーニング内容を記録する |
| 優先度 | 必須 |

**処理フロー:**
1. ユーザーが登録済みメニューを選択する
2. メニューに含まれる種目（MenuItems経由）がフォームに初期表示される
3. 各種目の重量・回数には前回のトレーニング記録値がデフォルトで入力されている（初回はExerciseの値を使用）
4. ユーザーがセットごとに実際の重量・回数を入力する
5. 必要に応じて種目を削除・追加する
6. クライアント側でZodによるバリデーション
7. データベースにTraining・ExerciseLog・ExerciseSetを保存
8. 記録した種目の登録種目マスタ（Exercise）の最新数値を更新する
9. 通算トレーニング回数（training_count）を自動カウントして記録する
10. 成功後、ホーム画面へ遷移する

**記録フィールド仕様（Trainingテーブル）:**

| フィールド名 | 必須/任意 | データ型 | バリデーション |
|-------------|----------|---------|---------------|
| ベースメニューID (base_menu_id) | **必須** | string | 登録済みメニューのID |
| トレーニング日時 (trained_at) | **必須** | datetime | デフォルトは当日 |
| 通算回数 (training_count) | **自動** | int | 自動カウント |

**種目記録（ExerciseLogテーブル）の各項目:**

| フィールド名 | 必須/任意 | データ型 | バリデーション |
|-------------|----------|---------|---------------|
| トレーニングID (training_id) | **必須** | string | トレーニング記録のID |
| 種目ID (exercise_id) | **必須** | string | 登録済み種目のID |

**セット記録（ExerciseSetテーブル）の各項目:**

| フィールド名 | 必須/任意 | データ型 | バリデーション |
|-------------|----------|---------|---------------|
| 種目記録ID (exercise_log_id) | **必須** | string | 種目記録のID |
| セット番号 (set_count) | **必須** | int | 1から連番 |
| 重量 (weight) | **必須** | int | 0以上、999以下 |
| 回数 (reps) | **必須** | int | 0以上、100以下（整数） |

#### FR-TRAINING-002: トレーニング記録時の種目編集

| 項目 | 内容 |
|------|------|
| 概要 | トレーニング記録中に種目を追加・削除する |
| 優先度 | 必須 |

**詳細要件:**
- メニューから読み込んだ種目を削除できる（その日はやらない種目を外す）
- 登録済み種目から追加できる（メニュー外の種目を追加する）
- 新規種目を入力して追加できる（種目マスタにも自動登録される）
- 種目の並び順を変更できる

#### FR-TRAINING-003: 前回記録のデフォルト入力

| 項目 | 内容 |
|------|------|
| 概要 | 前回のトレーニング記録を初期値として表示する |
| 優先度 | 必須 |

**詳細要件:**
- 各種目のセットごとの重量・回数に、前回の同一種目の記録値をデフォルトで入力する
- 前回記録は登録種目マスタ（Exercise）の最新数値（weight・reps）を参照する
- 該当種目の記録が存在しない場合は、Exerciseの値を使用する
- デフォルト値が入力された状態から、変更がある箇所のみユーザーが修正する

#### FR-TRAINING-004: 記録内容のコピー

| 項目 | 内容 |
|------|------|
| 概要 | トレーニング記録をテキスト形式でクリップボードにコピーする |
| 優先度 | 必須 |

**詳細要件:**
- 記録保存後に「コピー」ボタンを表示する
- ボタン押下でトレーニング記録をテキスト形式に整形し、クリップボードにコピーする
- コピー成功時にトースト通知を表示する
- テキストフォーマットは「8. コピー機能のテキストフォーマット例」に従う

**コピーテキストに含まれる情報:**
- 通算トレーニング回数（training_count）
- メニュータイトル（base_menu_id経由でMenuのtitleを取得）
- 種目ごとのセット数・重量（Exerciseから参照）
- セットごとの実際の回数・重量（ExerciseSetから取得）

#### FR-TRAINING-005: トレーニング履歴表示

| 項目 | 内容 |
|------|------|
| 概要 | 過去のトレーニング記録を一覧表示する |
| 優先度 | 必須 |

**詳細要件:**
- ホーム画面にトレーニング履歴を日付の降順で表示する
- 各記録は日付（trained_at）、メニュータイトル（base_menu_id経由）、通算回数（training_count）を表示する
- 記録をタップすると詳細を表示する
- 詳細画面からもコピーボタンを使用できる
- 無限スクロールによるページネーションを実装する（1回あたり20件）

#### FR-TRAINING-006: 登録種目の最新数値更新

| 項目 | 内容 |
|------|------|
| 概要 | トレーニング記録保存時に登録種目の最新数値を更新する |
| 優先度 | 必須 |

**詳細要件:**
- トレーニング記録を保存した際、記録に含まれる各種目の登録種目マスタを更新する
- 更新対象: 最新の重量・回数（最終セットの値、または最大重量のセットの値）
- 次回のトレーニング記録時にデフォルト値として使用される

## 7. 非機能要件

- SPAとして動作し、オフライン時の考慮はスコープ外
- レスポンシブ対応（主にスマートフォンでの利用を想定）

### 7.2 セキュリティ要件

#### NFR-SEC-001: 認証・認可

- Supabase Authenticationによる認証を必須とする
- Row Level Security（RLS）により、ユーザーは自身のデータのみアクセス可能
- APIキーはサーバーサイドで管理し、クライアントに露出させない

#### NFR-SEC-002: データ保護

- 通信はHTTPS（TLS 1.3）を使用する
- PostgreSQLのデータは保存時に暗号化される
- 個人情報の取り扱いはGDPR/個人情報保護法に準拠する
