# 開発ルール

## 共通ルール

日本語で対応すること

## セッション開始時の手順

[spec.md](/docs/spec.md) を読み込むこと。読んだら「📛spec.mdを読みました。」と表示すること

## モジュール構成

- /src: Next.jsのプロジェクト
  - /app: App Router
  - /components: 全画面共通で使用するコンポーネント
    - /ui: shadcnのコンポーネントがインストールされるディレクトリ。自作のコンポーネントは入れてはいけない
  - /lib: ライブラリの初期化
  - /utils: ユーティリティ

## 開発ルール

作業の最後にビルドが通ることを確認すること

```bash
$ pnpm build
```

## UI実装ルール

- 画面遷移するときはスケルトンを表示すること
  - 画面遷移したかどうかわからないとユーザーは不安になってしまいます
  - 画面遷移するときは一旦遷移先の画面を表示し、データ表示を待つ間はスケルトンを表示すること
  - Suspenseを使用し、静的な情報や枠組みは先に表示するようにすること

## Import パスエイリアス

- パスエイリアスは `@/` を使用すること（例: `@/components/ui/button`）
- `#/` は使用禁止。`package.json` の `imports` フィールドで `@/*` のみ定義されている
